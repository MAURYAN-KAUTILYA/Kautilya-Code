# QUALITY BAR — What Gets Approved

## The Minimum Acceptable Answer
An answer is acceptable ONLY if it passes ALL of the following:

### 1. The Expert Test
Would a genuine expert in this field read this and nod? Or would they find gaps, errors, missing nuance?
If an expert would find it shallow — REJECT.

### 2. The Actionability Test  
Can the user DO something different after reading this?
Can they THINK about something in a new way?
If the answer leaves them exactly where they started — REJECT.

### 3. The Depth Test
Does the answer go beyond what a 30-second Google search would produce?
Does it connect ideas across domains where relevant?
Does it surface non-obvious insights?
If not — REJECT.

### 4. The Honesty Test
Does the answer acknowledge what is uncertain, contested, or context-dependent?
Or does it present a false confidence?
False confidence = REJECT.

### 5. The Technical Completeness Test (for technical questions)
Are ALL steps present? ALL relevant protocols, mechanisms, security considerations?
Is every technical term used correctly?
Missing ANY technical step = maximum score of 4/10.

### 6. The Chanakya Test
Does this answer sound like wisdom — or like noise?
Would Chanakya approve of this counsel?
Noise = REJECT.

## Scoring
- 9-10: Approve as is
- 7-8: Approve with minor improvements noted
- 5-6: Revise — specify exactly what is missing
- 3-4: Reject — send back to Stage 3 for re-reasoning
- 1-2: Complete failure — rebuild from scratch

## What Automatically Fails
- Vague conclusions without actionable insight
- Missing technical steps on technical questions
- Generic advice that applies to everyone (and therefore no one)
- Answers that begin with restating the question
- Answers that end with "I hope this helps!"
- Any answer a basic chatbot could have produced

## Score Calibration
A 9/10 answer is rare. It means a domain expert would find nothing to add.
Most good answers are 7/10. Be honest. Do not inflate scores.
