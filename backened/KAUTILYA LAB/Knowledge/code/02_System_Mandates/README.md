# System Mandates

This folder defines the hard rules for how the backend should behave.

## Files

- `01_pipeline_rules.md` - pipeline rules and execution shape
- `02_king_directive.md` - the Rajarishi layer's governing brief
- `03_role_doctrine.md` - role responsibilities and execution limits
- `04_voice_overlays.md` - tonal and response overlays

## Why It Matters

These mandates keep the product disciplined. They decide what should happen, what should not happen, and how the system should speak while doing it.
