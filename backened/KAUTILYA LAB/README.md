# KAUTILYA LAB

The engine room of Kautilya Code. This folder powers the live server, the variant-aware pipeline, the session memory, and the tools that make the builder feel intelligent instead of decorative.

## Highlights

- `code-server.js` - main Express entry point and route host
- `pipeline/` - orchestration, command directives, specialist agents, and arbitration
- `infra/` - memory, session, runtime, relevance, checkpoint, and audit support
- `models/` - model and capability selection for the different product variants
- `shared/` - slash-command definitions shared with the frontend
- `Knowledge/` - doctrine, policies, prompts, and persisted session artifacts

## Reading Order

1. Start with [Knowledge](./Knowledge/README.md)
2. Then check [pipeline](./pipeline/README.md)
3. Then inspect [infra](./infra/README.md)
4. Finish with [models](./models/README.md) and [shared](./shared/README.md)

## Personality

This folder is where Kautilya gets its discipline: it knows when to route, when to remember, when to ask for approval, and when to keep the surface calm.
